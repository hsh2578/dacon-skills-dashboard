# Skills 기반 가치투자 종목 발굴 대시보드

DACON 월간 해커톤 · 투자 데이터를 시각화하라 — Skills 기반 대시보드 설계

---

## 1. 서비스 개요

### 1.1 어떤 서비스인가요

이 서비스는 한국 주식의 PER·PBR 시계열을 시장 지수와 개별 종목 단위에서 같은 기준으로 비교해 보여주는 대시보드입니다. 그 위에 4팩터 가치투자 시그널과 AI 정성 분석을 결합해 매주 살펴볼 만한 종목 10개를 자동으로 추출합니다. 시장이 지금 비싼지 싼지를 매크로 차트에서 먼저 확인한 뒤 개별 종목으로 좁혀 들어가는, 가치투자의 자연스러운 동선을 한 화면에서 마칠 수 있도록 설계했습니다.

대시보드를 만드는 분석 규칙 자체는 코드가 아니라 Skills.md 문서 한 장에 정의했습니다. 가중치, 필터, 시각화 방식이 모두 텍스트로 적혀 있어서 분석가가 .md 파일만 수정하면 결과가 바뀝니다. 이 점이 본 출품작의 가장 큰 정체성입니다.

### 1.2 왜 만들었나요

저는 평소 한국 주식의 PER·PBR이 5년 평균 대비 어디 위치해 있는지를 확인하면서 종목을 고르는 편입니다. 그런데 막상 자료를 찾아보면 이런 시계열을 정리해 놓은 사이트가 거의 없습니다. KOSPI나 KOSDAQ 지수의 PER을 일부 사이트가 제공하긴 하지만 갱신이 늦거나 평균 ±1σ 밴드 같은 맥락 정보가 빠져 있어서 "지금이 비싼지 싼지"를 한눈에 판단하기 어렵습니다. 개별 종목 단위로 들어가면 더 어려워집니다. 60개월 시계열을 같은 스케일로 그려 주는 곳이 사실상 없습니다.

이 빈 공간을 직접 메우는 것이 출발점이었습니다. 단순한 시계열 인프라에서 한 걸음 더 나아가, 객관적인 정량 시그널과 사업 모멘텀을 함께 보여 주는 정성 분석을 결합해 종목 발굴까지 자동화했습니다.

여기에 더해 분석 룰을 코드에 박아 두면 가중치 하나 바꿀 때마다 코드를 고쳐야 한다는 문제가 있었습니다. 이번 공모전 주제인 Skills.md는 이 문제를 정확히 풀어 줍니다. 룰을 .md 한 장에 정의하고 그 .md를 곧 알고리즘 명세서로 사용하는 방식입니다. 본 출품작은 이 패러다임을 실제로 구현한 사례입니다.

### 1.3 시장 갭과 본 서비스의 위치

| 영역 | 기존 한국 웹 | 본 서비스 |
|---|---|---|
| 시장 지수 PER·PBR 시계열 | 일부 제공, 갱신 늦음, 평균 밴드 부재 | KOSPI·KOSDAQ·S&P 500을 10년 시계열 + 평균 ±1σ 밴드로 표시 |
| 개별 종목 PER·PBR 시계열 | 사실상 없음 | 보통주 300여 종목을 60개월 시계열 + 평균 밴드로 통일 |
| 매크로와 마이크로 비교 | 별도 화면, 다른 단위 | 한 사이트 안에서 같은 스케일로 연속 조회 가능 |
| 분석 룰 정의 | 코드 안에 흩어진 if/else | Skills.md 한 장에 텍스트로 정의 |
| 정성 분석 | 별도 리서치 리포트 PDF | 출처 태그 강제된 AI 서브에이전트 자동 생성 |
| 외부 API 키 | 백엔드 운영이 필수 | 정적 호스팅, 키 노출 0 |
| 데이터 갱신 | 수동 ETL 파이프라인 | 슬래시 커맨드 한 줄 |

### 1.4 누가 쓰나요

처음에는 본인 사용을 염두에 두고 만들었습니다. 매주 한국 시장에서 5년 평균 대비 저평가된 종목을 찾는 개인 투자자 한 명입니다. 다음 단계로는 같은 룰을 자기 신념대로 가중치만 바꿔 쓰고 싶은 다른 투자자, 그리고 동일한 Skills.md 패턴을 ETF나 해외 시장 같은 다른 데이터 구조로 확장하려는 분석가까지 자연스럽게 확장될 수 있도록 구조화했습니다.

### 2.2 1단계 — Universe 정의

전체 KOSPI·KOSDAQ 보통주 약 2,300종목 중에서 분석 가능한 종목만 추립니다. 시가총액이 너무 작아 유동성이 부족하거나, 5년 평균 PER을 신뢰할 수 없거나, 컨센서스 데이터가 없는 종목은 분석 자체에서 제외합니다.

| 필터 | 임계값 | 제외 사유 |
|---|---|---|
| 시가총액 | 3,000억 원 이상 | 유동성 부족과 이상치 방지 |
| 종목 형태 | 보통주만 (우선주·리츠·스팩 제외) | PER 정의가 통일된 표본만 사용 |
| 5Y PER 검증 | 적자 분기가 4분기 이하 | 적자가 너무 많으면 5년 평균 PER이 의미를 잃음 |
| Forward PER | 보유 필수 | 4팩터 중 하나의 입력 |
| 컨센 평균 목표가 | 보유 필수 | 4팩터 중 하나의 입력 |

이 필터를 통과한 종목이 303개이며, 본 서비스의 모든 분석은 이 universe 안에서 이루어집니다.

### 2.3 2단계 — 4팩터 raw 계산

각 종목 단위로 네 개의 raw 시그널을 계산합니다. 정규화 이전 단계의 원본 값입니다.

| 팩터 | 정의 | 양수일 때 의미 | 그룹 |
|---|---|---|---|
| s1_per | (5Y 평균 PER − 현재 PER) ÷ 5Y 평균 PER | 자기 역사 대비 PER 저평가 | Value |
| s1_pbr | (5Y 평균 PBR − 현재 PBR) ÷ 5Y 평균 PBR | 자산가치 대비 PBR 저평가 | Value |
| s3_per | (현재 PER − Forward PER) ÷ 현재 PER | 향후 12개월 이익 개선 반영 | Growth |
| upside | (컨센 평균 목표가 − 현재가) ÷ 현재가 | 증권사 컨센 상승 여력 | Growth |

앞의 두 팩터는 과거 5년 평균 대비 현재 위치를 보고, 뒤의 두 팩터는 현재 대비 미래 위치를 봅니다. 두 시간축이 합쳐져야 "역사적으로 싼데 회복도 진행되고 있는" 종목을 골라낼 수 있다고 판단했습니다.

### 2.4 3단계 — Z-Score 정규화

처음에는 단순한 임계값 통과 방식을 썼습니다. "5년 평균보다 낮으면 통과, 아니면 탈락" 같은 식입니다. 그런데 평균 14.1과 현재 14.0은 통과시키고 14.2는 탈락시키는 절벽 효과가 발생했습니다. 종목 간 강도 차이가 사라지는 문제였습니다.

해결책으로 각 팩터를 universe 안에서 순위로 변환한 뒤 Z-Score로 변환하는 방식을 도입했습니다. 순위 기반 Z-Score는 outlier에 강하고 분포가 자동으로 정규화된다는 장점이 있습니다. 코드로 표현하면 다음과 같습니다.

```python
def zscore_from_rank(series):
    pct = series.rank(pct=True).clip(0.001, 0.999)
    return scipy.stats.norm.ppf(pct)

for factor in ["s1_per", "s1_pbr", "s3_per", "upside"]:
    df[f"{factor}_z"] = zscore_from_rank(df[factor])

df["value_score"]  = zscore(df["s1_per_z"] + df["s1_pbr_z"])
df["growth_score"] = zscore(df["s3_per_z"] + df["upside_z"])
df["total_score"]  = df["value_score"] * 0.60 + df["growth_score"] * 0.40
```

그룹 합산을 한 번 더 Z-Score로 재정규화하는 이유는 두 그룹의 분포 표준편차를 1로 맞추기 위해서입니다. 이렇게 하면 Value 그룹과 Growth 그룹의 점수를 같은 스케일에서 비교할 수 있습니다.

### 2.5 4단계 — 가중치와 티어링

가중치는 Value 60%, Growth 40%로 설정했습니다. 결정 근거는 신뢰도 차이입니다. 5년 평균 PER·PBR은 안정된 앵커가 있어 노이즈가 작은 반면, 컨센서스 추정치는 분기마다 변동이 큽니다. 신뢰도가 높은 쪽에 더 많은 비중을 두는 것이 합리적이라고 보았습니다.

이 6:4 가중치는 본 출품작의 결정값이지만, 동시에 사용자가 자기 신념대로 바꿀 수 있는 룰북 파라미터이기도 합니다. 메인 룰북 `value_recovery_quant.md`에 명시적으로 적혀 있어, .md만 수정하면 즉시 반영됩니다.

티어는 정규분포 기준의 표준 임계값을 따릅니다.

| 등급 | 임계값 | universe 상위 비율 |
|---|---|---|
| Hidden Gem | total_score ≥ +2.33σ | 1% |
| Strong Buy | total_score ≥ +1.65σ | 5% |
| Buy | total_score ≥ +0.84σ | 20% |
| Watch | total_score ≥ 0σ | 50% |
| Weak | total_score < 0σ | 하위 50% |

본 시점(2026-04-27 기준) Top 10 종목 중 9종목이 Strong Buy 이상에 해당합니다.

### 2.6 5단계 — AI 정성 분석

정량 시그널만으로는 "왜 이 종목이 지금 저평가되어 있는지"를 알 수 없습니다. 그래서 상위 10종목에 한해 AI 서브에이전트 3종이 자동으로 사업 모멘텀과 리스크를 생성합니다. 다만 정성 분석은 환각 위험이 크기 때문에 모든 항목에 출처 태그를 강제합니다.

| 태그 | 의미 | 사용처 |
|---|---|---|
| `[정량]` | 입력 JSON 필드에서 도출된 진술 | 정량 포인트 끝에 부착 |
| `[출처: WebSearch · YYYY-MM]` | WebSearch에서 직접 확인된 사실 | 정성 모멘텀·리스크 끝에 부착 |
| `확인 필요` | 출처가 모호하거나 1년 이상 오래된 정보 | 위험 신호 표시 |

출처 없는 주장은 룰 자체에 의해 출력 금지입니다. 이 구조 덕분에 와이지엔터테인먼트의 BABYMONSTER 컴백 일정이나 BIGBANG 20주년 같은 실시간 모멘텀이 환각 없이 카드에 들어갈 수 있습니다.

### 3.2 시장 지수 화면

첫 화면이 시장 지수인 이유가 있습니다. 개별 종목 PER이 5년 평균보다 낮아도 시장 전체 PER이 상단권이라면 시장 자체가 비싸다는 뜻입니다. 매크로 맥락을 먼저 본 다음 마이크로 종목으로 좁혀 들어가는 것이 가치투자의 자연스러운 동선입니다.

| 차트 | 데이터 출처 | 표시 |
|---|---|---|
| PER (시장) | KOSPI/KOSDAQ 일별 (pykrx), S&P 500 월별 (multpl) | 10년 시계열 + 평균선 + ±1σ 밴드 |
| PBR | 동일 출처 | 동일 |
| 배당수익률 | 동일 | 동일 |
| 종가 | 동일 | 정규화(=100) 토글 가능 |

차트 좌측 사이드바에는 현재 상태를 한 줄로 요약합니다. 예를 들어 "KOSPI PER 26.83 / 10년 평균 15.89 / +2.02σ"처럼 표시되어, 시장이 역사적으로 어디 위치해 있는지를 한눈에 알 수 있습니다.

## 4. Skills.md 설계 방향

### 4.1 표준 6섹션 스키마

본 프로젝트가 정의하고 따르는 모든 룰북은 동일한 6섹션 구조를 사용합니다. 이 구조가 곧 대시보드 자동 생성의 입력 명세서 역할을 합니다.

| 섹션 | 내용 |
|---|---|
| 1. 핵심 컨셉 | 한 줄 요약 + 어떤 시장 통찰에서 출발했는지 |
| 2. 데이터 소스 | 출처, 필드, 기간, 인증 여부 |
| 3. 1차 필터 | 통과하지 못하면 분석 자체에서 제외 |
| 4. 점수화 룰 | 정규화 방식, 가중치, 안전 가드 |
| 5. 시각화 룰 | 카드 형태, 색상, 차트 종류, 데이터 부족 시 표시 |
| 6. 인사이트 생성 | 자동 코멘트 템플릿, 3-소스 원칙 |

룰북을 추가하면 사이트의 새 섹션이 자동으로 매핑되도록 설계의 첫 단추를 마련했습니다. 사용자가 자기 신념을 담은 새 룰북을 작성해 추가하면, 기존 코드 수정 없이도 결과가 사이트에 반영될 수 있는 구조입니다.

### 4.2 룰북 카탈로그

제출 시점 기준 세 장의 룰북이 같은 universe와 같은 데이터 소스를 공유하면서 서로 독립적으로 정의되어 있습니다.

| 룰북 | 역할 | 가중치 구성 | 시각화 |
|---|---|---|---|
| value_recovery_quant.md (메인) | 4팩터 가치+성장 결합 | Value 60 + Growth 40 | Top 10 카드 + 시그널 막대 |
| forward_attractive.md (보조) | Forward PER 단독 매력 | 단일 팩터 | 산점도 (Fwd PER × 시총) |
| dividend_value.md (보조) | 인컴 + 가치 결합 | Income 50 + Value 50 | 듀얼 축 차트 |

같은 스키마가 서로 다른 투자 철학을 어떻게 표현할 수 있는지 보여 주는 것이 의도입니다. 가중치를 다르게 두는 것만으로 "회복 모멘텀 강세"부터 "안정 인컴 우선"까지 표현 가능합니다.

### 4.3 정성 분석도 Skills.md입니다

본 출품작에서는 AI 서브에이전트도 Skills.md 형식으로 정의했습니다. `skills/agents/` 폴더 안에 세 개의 .md 파일이 있습니다.

| 에이전트 | 담당 | 출처 태그 |
|---|---|---|
| fundamental_analyst | 사업 모멘텀 (단기 촉매, 구조적 드라이버, 최근 실적) | [출처: WebSearch · YYYY-MM] |
| risk_analyst | 하방 리스크 (실행, 시장 인식, 경쟁, 거시) | 동일 |
| synthesizer | 두 결과를 카드 한 장 분량으로 압축 | 출처 태그 절대 제거 금지 |

세 에이전트 모두 동일한 출처 태그 강제 룰을 따릅니다. 어떤 항목이든 "이 정보가 어디서 왔는지"를 명시하지 않으면 출력에서 막힙니다. 환각을 차단하는 가장 단순하고 강력한 장치라고 판단했습니다.

이 구조 덕분에 정성 분석 자체가 재현 가능한 룰북이 됩니다. 새 종목, 새 시점에도 동일한 구조의 분석이 자동으로 생성됩니다. Anthropic 공식 형식에 맞춘 실행용 버전은 `.claude/agents/`에 별도로 두었으며, 내용은 동일하고 형식만 다릅니다.

### 4.4 3-소스 원칙

본 서비스의 모든 룰북은 세 가지 출처만 인용할 수 있습니다.

| 소스 | 적용 영역 | 태그 |
|---|---|---|
| 사전 페치 정량 | pykrx, 네이버 컨센, multpl, yfinance에서 페치된 JSON | [정량] |
| Tool Call 결과 | WebSearch (정성 룰북에서만 사용) | [출처: WebSearch · YYYY-MM] |
| 룰 정의 자체 | Skills.md에 적힌 임계값, 가중치, 공식 | 표기 불필요 |

이 외의 출처는 금지입니다. 특히 LLM이 학습 단계에서 외운 수치를 인용하면 환각으로 분류되어 출력에서 제외됩니다. 단순한 룰이지만, 모든 정성 인사이트의 신뢰성을 담보하는 가장 중요한 안전장치입니다.

## 5. 바이브코딩 활용

### 5.1 슬래시 커맨드 한 줄로 운영

데이터 갱신 작업은 Claude Code의 슬래시 커맨드 한 줄로 실행됩니다. `.claude/commands/update-data.md` 한 장이 7단계 파이프라인 전체를 정의합니다.

| 명령 | 동작 | 소요 시간 |
|---|---|---|
| /update-data | 시장 지수 + 매트릭스 + 점수 + 일별 풀 + AI 분석 | 약 12분 |
| /update-data --full | 위 + universe 재구축 | 약 30분 |
| /update-data --skip-ai | AI 분석 건너뛰고 정량만 | 약 10분 |

### 5.2 7단계 파이프라인

| 단계 | 스크립트 | 출력 |
|---|---|---|
| 1. 시장 지수 | fetch_kr.py + fetch_us.py (병렬) | kospi.json, kosdaq.json, sp500.json |
| 2. Universe 재구축 | fetch_universe → validate → fetch_consensus | universe.json |
| 3. PER·PBR 매트릭스 | fetch_matrix.py | matrix/{per,pbr}_monthly.json |
| 4. Top 10 점수 | apply_triple_cross.py | screens/triple_cross.json |
| 5. Top 10 일별 풀 | fetch_batch.py --top-n 10 | stocks/{code}.json × 10 |
| 6. AI 정성 분석 | Agent tool × 10 (병렬) | screens/ai_notes.json |
| 7. 완료 보고 | 마크다운 표 자동 생성 | (사용자에게 표시) |

각 단계가 모두 스크립트로 정의되어 있고, Claude Code가 단계별로 순차 실행합니다. 사용자는 결과를 GitHub에 commit하기만 하면, 약 30초 만에 GitHub Pages로 자동 배포됩니다.

### 5.3 코드 작성 비중

이번 프로젝트의 코드는 거의 전부 Claude Code와의 대화로 생성되었습니다. 제가 직접 작성한 부분은 다음으로 한정됩니다.

| 영역 | 직접 작성 |
|---|---|
| 룰북·기획서 검토와 결정 | 가중치 6:4, 시가총액 컷 3,000억, 5년 평균 기간 등 |
| 파이프라인 주요 결정 | 정규화 vs hard rule, Forward PER 추가, 매트릭스 fallback 분리 등 |
| 디자인 톤·레이아웃 피드백 | 네이버 금융 스타일, 한국 색 컨벤션, 이모지 배제 등 |

코드 자체는 LLM이 쓰고, 사용자는 룰과 결정에 집중하는 방식입니다. Skills.md를 잘 정의하면 코드는 LLM이 쓴다는 본 공모전의 주제와 정확히 부합합니다.

### 5.4 정적 호스팅으로 심사 규칙 100% 준수

대회 규칙 중 "외부 API 사용 시 심사자가 별도 키 없이 확인 가능해야 한다"는 조항이 본 서비스의 가장 큰 설계 결정을 강요했습니다.

| 항목 | 본 서비스 처리 |
|---|---|
| 백엔드 | 없음 |
| API 키 사이트 노출 | 0 (모든 키는 로컬 .env에만) |
| 데이터 갱신 후 반영 | 커밋 후 약 30초 (GitHub Pages 자동 배포) |
| 캐시 무효화 | 사이트가 fetch 시 ?t={타임스탬프} 자동 부착 |
| 별도 빌드 단계 | 없음 (정적 파일 그대로 서빙) |

심사자는 URL 한 줄로 즉시 접속할 수 있고, 어떤 인증이나 설정 없이 모든 기능을 그대로 확인할 수 있습니다.

## 6. 차별화 포인트

다른 출품작과 본 서비스를 구분 짓는 다섯 가지를 정리하면 다음과 같습니다.

### 6.1 PER·PBR 시계열 인프라 자체가 시장 갭을 메웁니다

한국 웹에서 KOSPI·KOSDAQ과 개별 종목의 PER·PBR을 같은 스케일로 비교 가능한 자료를 찾기 어렵다는 문제에서 출발했습니다. 시장 지수 60개월과 개별 종목 매트릭스(303종목 이상) 모두 동일한 평균 ±1σ 밴드 시각화로 통일했습니다. 매크로 맥락을 본 다음 마이크로 종목으로 들어가는 가치투자의 본연 동선을 한 화면 안에서 마칠 수 있습니다.

### 6.2 분석 룰을 .md 한 장에 정의하는 패러다임을 실제로 구현했습니다

가중치, 필터, 정규화 방식이 모두 텍스트로 적혀 있고, 그 텍스트가 곧 알고리즘 명세서입니다. 사용자가 정의한 새 룰북을 추가하면 사이트의 새 섹션으로 자동 매핑되도록 설계의 첫 단추를 마련했습니다. 룰을 바꾸는 행위가 코드 수정이 아니라 .md 편집이 되는 변화입니다.

### 6.3 정성 분석 자체도 Skills.md입니다

AI 서브에이전트 세 종류가 Top 10 종목에 대해 사업 모멘텀과 리스크를 자동 생성하며, 모든 항목에 출처 태그를 강제합니다. 출처 없는 주장은 룰에 의해 출력 금지입니다. 이 구조 덕분에 BABYMONSTER의 'CHOOM' 발매와 6월 서울 월드투어, BIGBANG 8월 20주년 컴백 같은 실시간 사업 모멘텀이 환각 없이 카드에 들어갈 수 있습니다.

### 6.4 정적 호스팅으로 심사 규칙을 100% 준수합니다

백엔드는 없습니다. API 키 사이트 노출은 0입니다. 모든 데이터는 로컬에서 사전 페치된 후 JSON으로 commit되며, GitHub Pages가 정적 파일을 그대로 서빙합니다. 심사자는 URL 한 줄로 즉시 접속할 수 있고, 데이터 갱신은 commit 후 30초 이내에 자동 반영됩니다.

### 6.5 슬래시 커맨드 한 줄로 전체 운영이 가능합니다

`.claude/commands/update-data.md` 한 장이 시장 지수, universe, 매트릭스, 점수, 일별 풀, AI 정성 분석 7단계 파이프라인을 정의합니다. 사용자는 채팅창에 `/update-data`만 입력하면 12분 만에 전체 데이터가 오늘 자로 갱신됩니다. 분석 룰을 .md로 정의했듯이 운영 워크플로우 자체도 .md로 정의되는 일관성이 본 출품작의 정체성을 형성합니다.

## 8. 결론

본 출품작은 "투자 분석 룰을 .md 한 장으로 정의하고, 그 룰을 다양한 데이터 구조에 자동 매핑해 대시보드를 생성한다"는 단일 명제를 처음부터 끝까지 관통합니다.

분석 룰은 `value_recovery_quant.md`가 4팩터 Z-Score 가중합산을 정의하고, 데이터 적재는 `/update-data` 슬래시 커맨드가 7단계 파이프라인을 자동 실행합니다. 시각화는 3개 섹션의 정적 사이트가 룰 출력을 카드와 차트로 표시하며, 정성 분석은 AI 서브에이전트 세 종류가 출처 태그가 붙은 인사이트를 자동 생성합니다. 재현성은 정적 호스팅과 JSON 커밋으로 누구나 동일한 결과를 확인할 수 있도록 보장합니다.

평가 5축에서 모두 명확한 정량 근거를 갖춘 출품작을 목표로 했습니다. 한국 웹에 비어 있던 PER·PBR 시계열 인프라를 메우면서, 동시에 Skills.md 패러다임의 실제 구현 사례를 제시하는 것이 본 출품작의 핵심 의도입니다.
